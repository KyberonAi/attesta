# Support Policy

This repository is maintained on a best-effort basis for open-source users.

## Channels

- **Bug reports / feature requests**: GitHub Issues
- **Security vulnerabilities**: GitHub Security Advisory (private)
- **Contributions**: Pull Requests

## Response Targets

- New issues: triage label and first maintainer response within 5 business days
- Pull requests: first review or status update within 5 business days
- Security reports: acknowledgment within 3 business days (see `SECURITY.md`)

## Scope

Community support covers:

- `python/` and `typescript/` core SDK behavior
- Public docs under `docs/`
- OSS no-code connectors in `packages/`

Community support does **not** include:

- Custom enterprise integration work
- Guaranteed response SLAs
- Incident-response operations for third-party deployments

## Version Support Window

- Latest `main`
- Latest published `0.1.x`

Older versions may receive guidance, but fixes are not guaranteed.

## Escalation

If an issue is blocked for more than 10 business days without a maintainer update, open a new issue tagged `maintainer-escalation` and link the original thread.
